Polygon Collision Detection
25 Apr 2016

The simplest technique used to detect a polygon collisions is SAT (Separating Axis Theorem), in this technique each edge of the polygon is treated as an separate axis.
Just like how the x and y axis have their own direction, by separating the axis we want to represent the direction that is normal (perpendicular) to each edge as a new axis in variables that we call the unit vector.

Once we know the unit vector we can then calculate the distance between the edge and the point of interest against the new axis. A line representing this distance can be seen below. Notice how the line is always perpendicular to the edge.

After interacting with the diagram you may have realised that a collision occurs only when each of the edges have a distance that is smaller then or equal to zero.

Advantages

This algorithm allows for a fast detection of fail cases, as soon as an edge is found to have a distance larger then zero the algorithm can immediately be stopped since we know there can't be a collision no matter what the other edge's distances may be. This of course saves a lot of computation time since the remaining edges don't have to be analysed.

The performance could be improved even further with the integration of voronoi regions.

Limitations

SAT only works with convex polygons, this should be kept in mind before being integrated in any project.
